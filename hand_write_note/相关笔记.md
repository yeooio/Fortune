# 相关笔记

## 一、手写步骤还原

1. 以普通用户身份接入目标系统。
2. 利用请求头 URL 下载功能的 SSRF 漏洞，通过 `file://` 协议读取服务端 `/etc/redis/redis.conf` 配置文件。
3. 从获取的 `redis.conf` 中提取 Redis 服务的主机地址、端口、访问密码等核心配置。
4. 利用 `urllib` 请求头注入漏洞，注入 Redis 协议命令，读取目标应用存储在 Redis 中的 `secret_key`（Flask 应用核心密钥）。
5. 使用 `flask-session-cookie-manager` 工具，结合泄露的 `secret_key`，伪造管理员 `admin` 用户的 session cookie。
6. 携带伪造的 `admin` cookie，访问后台 `/ad/users` 路由，越权获取系统全量用户信息。

## 二、进阶链路复盘（Flask + Redis）

这是上一阶段 Flask + Redis 应用渗透的进阶 RCE（远程命令执行）攻击链，完整步骤如下：

1. 核心前置条件：已通过 SSRF 漏洞获取 Redis 服务的地址、端口、密码等访问凭证（来自上一阶段信息收集）。
2. 攻击核心思路：通过 SSRF 向 Redis 写入恶意 Python 序列化数据，触发目标应用反序列化漏洞执行系统命令，再通过 SSRF 读取命令执行结果，完成完整 RCE 闭环。
3. 分步执行流程：

- 步骤 1：构造恶意 payload。
- 核心执行命令：`ps -ef > /tmp/output`（查看系统进程，并将结果写入临时文件）。
- 前置处理：对 payload 进行 URL 编码，适配 SSRF 的协议传输要求。

- 步骤 2：通过 SSRF 发送 payload，写入 Redis。
- 利用 SSRF 的 Gopher 协议，构造完整的 Redis 写入命令，将恶意序列化数据写入 Redis 的指定 Key（目标接口会读取的 Key）。

- 步骤 3：访问目标接口，触发反序列化漏洞。
- 访问 `/admin/online-users` 接口，该接口会从 Redis 读取数据并直接执行反序列化操作，触发恶意 payload，执行系统命令，将结果写入 `/tmp/output`。

- 步骤 4：通过 SSRF 读取命令执行结果。
- 再次利用 SSRF 的 `file://` 协议，读取服务端 `/tmp/output` 文件，获取系统命令执行结果，完成攻击闭环。
